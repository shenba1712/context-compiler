Limits & free-tier notes

This is a public free-tier demo — no accounts, shared capacity. Limits below are enforced per IP on this server; provider free tiers add their own caps on top.

  • Compile once and expand are free of model cost (local BM25). They share a pool of about 30 requests every 5 minutes.
  • Prove answer parity checks your Compile result (not Agent) using free-tier LLM keys. Each prove costs 4 of those points — roughly 7 proves per 5 minutes. The full-file side is capped at about 60,000 tokens.
  • Run agent also uses those free-tier keys and is heavier: 8 points each — roughly 3 runs per 5 minutes. After a run you can optionally Compare to full file (same cost as Prove). At most 2 LLM jobs run at once on this host.
  • If Gemini/OpenRouter free quota is exhausted, Prove and Agent may return 429/503 even when our rate limit still has room — wait a few minutes and retry.

Use it in your own AI agent

  
The demo above runs the exact same compile_context tool this server exposes — so what you
    just saw is the MCP tool's output. In real use it's an MCP server your coding agent calls automatically,
    so it reads the 3% that matters instead of dumping whole files into the model.


  

    
Without Context Compiler

      
Your agent needs one fact from a 40-page PDF, so it loads the entire file into the model —
        thousands of tokens, most of them irrelevant, paid for on every turn.

    

    
With Context Compiler

      
Your agent calls compile_context(file, task, budget) and gets back only the relevant
        sections under a token budget. Same answer, a fraction of the tokens — and it can
        expand_section if it needs more.

    

  


  
Where it connects: it's a local process your agent launches over stdio.
    Add one entry to your client's MCP config, point CC_ROOT at the folder the agent may read, and the two
    tools appear automatically. No API key needed for compilation.


  

    
OpenAI Codex~/.codex/config.toml
      

    
[mcp_servers.context-compiler]
command = "node"
args = ["/path/to/context-compiler/dist/server.js"]

[mcp_servers.context-compiler.env]
CC_ROOT = "/path/agents/may/read"


    
Claude Desktop / CursorJSON config
      

    
{
  "mcpServers": {
    "context-compiler": {
      "command": "node",
      "args": ["/path/to/context-compiler/dist/server.js"],
      "env": { "CC_ROOT": "/path/agents/may/read" }
    }
  }
}


    
Claude Codeone command
      

    
claude mcp add context-compiler -- node /path/to/context-compiler/dist/server.js

  


  

    
compile_context(file_path, task, token_budget=4000)
      → the compiled markdown, savings stats, and a manifest of what was left out.

    
expand_section(file_path, section_id, token_budget=2000)
      → one omitted section by id, when the agent decides it needs more.
